# 第三阶段确认记录：UI、命名、去重与原生 4K

状态：已实现并进入自动化与 EXE 验证。

## 已锁定并实现

- 默认窗口由 1000×650 再等比例缩小约 10% 为 900×585，最小尺寸 820×540。
- 当前壁纸预览、信息卡片、边距和字号同步缩小。
- Wallpaper 库列数根据可用宽度实时重排，不再固定 3 列。
- 紧凑模式在默认窗口目标为 6 列；窗口放大后可增加到 8 列。
- 提供四级缩略图大小：紧凑、中、大、特大；默认 6 列，放大后约 4–3 列。
- 项目预览从裁切填满改为完整比例显示，允许留白，不裁掉人物或画面边缘。
- 每次先加载 48 项；滚动接近最后一排时自动加载下一批。
- 当前快照保存前读取 Wallpaper Engine 当前项目路径，并匹配 Workshop ID 与 `project.json.title`。
- 当前快照、自动保存、视频帧和 Scene/Web 快照均优先使用原始标题。
- 成功与重复操作改用窗口底部临时提示，约 4.2 秒后消失；失败仍显示明确错误对话框。
- 去重同时使用历史 SHA-256 与目标目录实际文件 SHA-256；清空历史也不会重复保存相同内容。
- Wallpaper Engine 动态预览取消 `activate` 抢焦点参数；新增关闭按钮和主窗口 Esc 关闭命令。
- “检查原生 4K”只使用真实原始来源，不对 2K 快照做虚假软件放大。

## 原生 4K 边界

- 当前快照若已达到 3840 像素长边，直接按原始格式保存。
- 当前项目若是视频，进入原视频选帧器；只有视频源真实达到 4K 才能保存原生 4K 帧。
- 当前项目若是原始静态图，只有该图真实达到 4K 才直接保存。
- Scene/Web 的最终画面由 Wallpaper Engine 实时合成；2K 屏生成的 2560×1440 快照不能恢复不存在的 4K细节。
- 本程序不会将插值放大称为“更清晰”或“画质提升”。
