# Wallpaper Engine 壁纸导出助手——需求确认书 v0.1

状态：已由用户于 2026-07-20 确认，进入代码实现。

确认选择：Q1-A、Q2-A、Q3-A、Q4-A、Q6-A；Q5 采用 A+B 融合。由于当前会话未提供 `imagegen` 内置工具，A 以程序内原创 Qt Q 版小助手实现，B 使用用户提供参考图的本地裁切素材实现；没有切换到需要 API 密钥的 CLI 路径。

新增画质硬约束：保留源文件真实格式与原始像素，不放大、不二次压缩；若同一来源存在多个可用候选，优先实际像素面积最大的版本，4K 高于 2K。JPG 原样保存为 JPG，PNG 原样保存为 PNG。界面必须如实显示源分辨率，不能把格式转换或软件放大描述为画质提升。

## 1. 项目目标

制作一个适用于 Windows 11 的中文图形化 `.exe`，帮助用户把 Wallpaper Engine 生成或保存的可用静态图片安全地导出到自定义文件夹，并可持续归档后续出现的新壁纸快照。

程序只读取 Wallpaper Engine、Steam Workshop 和 Windows Themes 中的源文件；导出时只复制或转换到用户指定位置，不移动、不删除、不修改任何源文件。

## 2. 必须先区分的三类内容

### A. 当前桌面快照

来源通常是：

`%APPDATA%\Microsoft\Windows\Themes\WallpaperEngineLockOverride.jpg`

这是当前动态壁纸渲染后得到的一张无桌面图标静态图。程序可以稳定地复制、转为 PNG、重命名、去重和持续监控。

### B. Windows 主题缓存中的其他图片

可能包括：

- `WallpaperEngineOverride.jpg`
- `WallpaperEngineBackupWallpaper.jpg`
- `TranscodedWallpaper`
- `CachedFiles` 目录中的缓存图片

这些文件数量有限，可能陈旧或重复，不能等同于“过去使用过的所有 Wallpaper Engine 壁纸”。程序可以识别有效图片后批量导出，并标明来源。

### C. Steam Workshop 已安装项目

默认目录通常是：

`C:\Program Files (x86)\Steam\steamapps\workshop\content\431960`

其中可能包含视频、网页、应用程序和 `scene.pkg` 场景。只有项目文件夹里真实存在的 JPG/PNG 等图片才能直接复制；视频和场景的最终画面并不一定存在为一张原图。

因此，若用户所说的“全部静态壁纸”是指全部 Workshop 项目，必须约定：导出封面预览、导出项目内所有图片素材，还是只导出能够确认是完整壁纸的图片。三者结果完全不同。

## 3. 建议的第一版功能范围

### 3.1 首页 / 当前壁纸

- 自动检测当前快照文件。
- 显示大图预览、分辨率、格式、文件大小和更新时间。
- 选择目标文件夹。
- 一键“导出当前壁纸”。
- 可选导出为原始 JPG 或无损 PNG。
- 自动生成不会覆盖旧文件的名称，例如 `WP_2026-07-20_14-30-25.png`。
- 按图片内容哈希去重；相同图片不重复保存，并明确提示。
- “打开导出文件夹”和“复制源文件路径”。

### 3.2 批量导出

- 扫描 Windows Themes 中实际存在且能被验证为图片的候选文件。
- 列表显示缩略图、来源、尺寸、更新时间和是否已导出。
- 支持全选、取消选择和批量导出。
- 对损坏文件、重复图片或尺寸异常项目给出中文说明。

Workshop 扫描是否纳入第一版，等待用户确认第 7 节的问题。

### 3.3 一键更新文件夹

- 手动点击“立即检查更新”。
- 将自上次成功归档后出现的新快照复制到指定文件夹。
- 使用内容哈希而不是只看文件名或修改时间，避免重复导出。
- 本地保存一份导出记录，记录源路径、哈希、发现时间、导出时间和目标路径。
- 用户可更换保存位置；新任务使用新位置，旧文件不会被自动搬迁。

### 3.4 自动监控（建议功能，需确认）

- 程序运行时监控当前快照是否变化。
- 检测到真正的新图片后自动归档。
- 可选最小化到系统托盘。
- 默认不随 Windows 开机启动；若需要，可由用户在设置中明确开启。
- 程序退出后不再监控，除非用户确认需要后台常驻或开机自启。

### 3.5 按时间筛选导出

可支持两种日期依据，但必须选定默认语义：

1. “Wallpaper/Windows 写入源文件的修改时间”；适合首次扫描现有文件。
2. “本程序首次发现或成功归档的时间”；更可靠，适合长期归档。

界面提供开始日期、结束日期、预览匹配数量、选择目标文件夹和执行导出。结束日期按当天 23:59:59 计算。

### 3.6 历史记录

- 展示成功、跳过重复和失败的记录。
- 支持按日期和状态筛选。
- 支持打开图片、打开所在文件夹。
- 允许清除“程序记录”，但默认绝不删除已导出的图片；若未来增加删除功能，必须二次确认。

### 3.7 设置与诊断

- 自动检测 Themes 路径与 Wallpaper Engine Workshop 路径。
- 支持手动指定非默认 Steam 库路径。
- 选择默认导出目录和默认格式。
- 显示清晰的中文运行日志。
- “重新扫描”“打开源目录”“打开日志目录”。
- 所有配置和记录保存在用户本地，不上传网络。

## 4. UI 与视觉方向

- 整体风格：清爽、柔和、偏二次元的粉色 + 天蓝色 + 白色。
- 主色建议：樱花粉 `#F4AFC2`、浅海蓝 `#A9DDF2`、深蓝灰文字 `#44536B`、乳白背景 `#FFF9FC`。
- 布局建议：左侧圆角导航栏；右侧为当前壁纸大预览、状态卡片和主要操作；底部固定显示扫描/导出进度。
- 页面：当前壁纸、批量导出、更新归档、历史记录、设置。
- 动效保持轻量，避免影响扫描和导出速度。
- 不直接把五张参考壁纸整张铺成界面背景，以免文字看不清。
- 可在顶部横幅、空状态和侧栏放置 Q 版头像或角色小图，但素材方案等待用户确认：直接使用用户给的图片裁切，或重新生成可随程序分发的原创 Q 版素材。
- 应支持 125%、150%、200% Windows 缩放，窗口最小尺寸下不遮挡主要按钮。

## 5. 建议的安全与质量规则

- 永远不覆盖同名文件；名称冲突时自动加序号。
- 导出采用临时文件写入后再改名，避免中断产生半张图片。
- 复制后验证目标文件可以被识别为图片。
- JPG 转 PNG 不会提高原图清晰度；界面应如实提示。
- 扫描时忽略视频、可执行文件、压缩包、`scene.pkg` 等不可直接当作完整静态图的内容。
- 大目录扫描放在后台线程，UI 不假死，可取消。
- 不需要管理员权限；若某个源目录无权读取，只提示用户，不尝试修改系统权限。
- 不联网、不上传图片、不修改 Wallpaper Engine 或 Steam 文件。

## 7. 等待用户确认的关键问题

### Q1：“全部静态壁纸”具体指哪一范围？

- 方案 A（推荐）：当前快照 + Windows Themes 中现存的其他有效图片；结果最可靠。
- 方案 B：再扫描全部 Workshop 项目，只导出其中能确认存在的 JPG/PNG；可能包含预览图、贴图和零散素材，并不都是真正完整壁纸。
- 方案 C：A 与 B 都做，但 Workshop 结果单独分类为“项目图片素材”，不宣称都是完整壁纸。

### Q2：“自动保存新增壁纸”以什么事件为准？

- 方案 A（推荐）：`WallpaperEngineLockOverride.jpg` 内容变化，即当前桌面换成一张新壁纸快照。
- 方案 B：Steam 新订阅/下载了一个 Workshop 项目。
- 方案 C：两者都监控，分别归类。

### Q3：自动更新的运行方式？

- 方案 A（推荐）：程序打开时自动监控，关闭后停止；同时保留“立即检查更新”。
- 方案 B：最小化到托盘持续监控，用户从托盘退出。
- 方案 C：增加开机自启并后台常驻。

### Q4：默认导出格式？

- 方案 A（推荐）：保留原格式；用户需要时单独选择转 PNG，速度快且不增大文件。
- 方案 B：所有图片统一转 PNG；文件通常更大，但格式统一。
- 方案 C：同时保存原图和 PNG；占用空间最多。

### Q5：界面角色图片如何处理？

- 方案 A（推荐）：根据粉蓝海洋/樱花气氛生成原创 Q 版角色素材，随 `.exe` 使用。
- 方案 B：直接裁切并使用用户发来的五张参考图；需要用户确认其拥有用于本程序的使用权限。
- 方案 C：第一版只用原创图标、花瓣和气泡装饰，不嵌入人物。

### Q6：“按某个时间段之后更新”采用哪个时间？

- 方案 A（推荐）：本程序发现/归档时间；从安装程序之后开始最准确。
- 方案 B：源文件最后修改时间；可筛选首次扫描前已经存在的内容，但时间可能被 Steam 或系统改写。
- 方案 C：界面允许二选一，默认使用发现时间。
