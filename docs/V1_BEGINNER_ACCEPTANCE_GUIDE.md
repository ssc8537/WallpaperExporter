# V1 小白验收教程

这份教程只检查你最后反馈的修改，不要求重新测试已经默认通过的旧功能。

自动化检查已经通过 25 项，但自动化测试不能代替真实 Wallpaper Engine、桌面切换、Steam 页面和全局快捷键验收。请按下面顺序操作。

## 1. 使用正确的 V1 程序

1. 先关闭旧版 WallpaperExporter。
2. 找到 `dist\WallpaperExporter_V1.zip`。
3. 右键 ZIP，选择“全部解压缩”。
4. 打开解压后的文件夹，双击 `WallpaperExporter.exe`。
5. 不要把 EXE 单独拿走；旁边的 `_internal` 文件夹必须保留。

通过标准：程序大约几秒内显示窗口，不再像旧单文件版一样长时间等待解压。

失败时请记录：从双击到窗口出现大约用了多少秒。

## 2. 检查左侧目录和最小窗口

1. 保持程序刚打开时的默认大小，不要先最大化。
2. 左侧应同时看到：
   - 当前壁纸
   - Wallpaper 库
   - Themes 批量
   - 更新与历史
   - 设置
   - 桌面全局快捷键
3. 查看左下角，“图库：已加载/等待扫描”进度和“4K 优先 · 不压缩 · 不放大”应紧挨在一起。
4. 六个页面标签不应被进度条遮挡。

通过标准：六个页面都能直接点击，左下角没有大块无意义空白把图库进度和 4K 说明隔开。

## 3. 检查桌面全局快捷键页面

1. 点击左侧“桌面全局快捷键”。
2. 不滚动页面，确认能直接看到：
   - 保存当前最高画质壁纸
   - 切换下一张壁纸
   - 返回上一张壁纸
   - 在 Wallpaper 中打开 / 管理 / 取消订阅
   - 打开设置
   - 打开更新与历史
3. 点击“打开设置”，确认进入设置页。
4. 返回快捷键页，点击“打开更新与历史”，确认进入历史页。
5. 打开 Wallpaper 库，确认每张图库卡片上不再有重复的“Steam 管理”按钮。

通过标准：当前桌面项目的管理入口只集中在“桌面全局快捷键”页面。

## 4. 检查 Steam 管理当前壁纸

1. 先在桌面确认当前正在播放哪张壁纸。
2. 回到程序的“桌面全局快捷键”页面。
3. 点击“在 Wallpaper 中打开 / 管理 / 取消订阅”。
4. 等待 Steam 打开 Workshop 页面。
5. 对照页面标题，确认它是当前桌面项目，而不是之前的壁纸。

说明：程序负责打开准确的 Workshop 页面。真正取消订阅仍由已登录的 Steam 页面确认，程序不会删除本地文件冒充取消成功。

## 5. 检查当前壁纸刷新和旧快照保护

1. 打开程序的“当前壁纸”页面，记下“当前 Wallpaper 项目”的名字。
2. 在 Wallpaper Engine 中切换到另一张壁纸。
3. 等待 2 至 5 秒，然后回到程序点击“刷新”。
4. 检查“当前 Wallpaper 项目”的名字是否已经变成新项目。

刷新后可能出现两种正确结果：

### 结果 A：Wallpaper Engine 已生成新快照

- 预览和更新时间都更新。
- 点击保存后得到新壁纸。
- 再保存一次会提示重复，并且不会多出第二份相同文件。

### 结果 B：Wallpaper Engine 尚未生成新快照

- 项目名称已经变成新项目。
- 界面用项目预览帮助识别。
- 蓝色说明明确写着“尚未写出新的高画质快照”。
- Scene/Web 点击保存时，程序阻止保存旧图。
- 视频项目点击保存时，程序打开原视频选帧器。
- 直接图片项目使用项目原始图片保存。

结果 B 不是“刷新仍然失效”，而是 Wallpaper Engine 本身没有更新 Windows Themes 快照。若要测试新快照，请在 Wallpaper Engine 设置中启用类似“覆盖锁屏图像 / Override lock screen image”的选项，再切换壁纸并等待更新。不同语言版本的文字可能略有不同。

真正失败：桌面已经换了壁纸，但点击刷新后，“当前 Wallpaper 项目”的名字仍然是旧项目。

## 6. 检查最近上一张逻辑

请慢一点操作，每次切换后等待桌面完成变化：

1. 记住开始壁纸，称为 A。
2. 点击“下一张”，等变化完成，称为 B。
3. 再点“下一张”，等变化完成，称为 C。
4. 再点“下一张”，等变化完成，称为 D。
5. 点击“返回上一张”。正确结果应该是 C，而不是 A。
6. 再点击一次“返回上一张”。正确结果应该是 B。
7. 再点击“下一张”。正确结果应继续从最近位置切换，不应固定跳回最开始的 A。

然后用默认全局快捷键重复一次：

- 下一张：`Ctrl + Alt + 右方向键`
- 上一张：`Ctrl + Alt + 左方向键`
- 保存：`Ctrl + Alt + S`

真正失败：连续切换多张后，第一次按上一张仍固定回到最开始的 A。

## 7. 检查启动与图库延迟加载

1. 完全退出程序，再重新打开。
2. 启动时不应立即因为扫描数百张 Workshop 壁纸而长时间卡住。
3. 第一次点击“Wallpaper 库”后才开始读取图库。
4. 左下角进度会从“等待扫描”变成“已加载 数量/总数”。

说明：第一次进入图库仍需要读取项目和预览图，这是正常加载；程序启动本身不再等待整库扫描。

## 8. 动态预览已知限制

1. 在图库里选择一个 Scene/Web 项目并打开动态预览。
2. 动态窗口可能需要短暂加载，鼠标也可能短时间显示忙碌。
3. 按 Esc 或点击“关闭动态预览”应能关闭窗口。

短暂加载目前记录为已知限制，不作为本次失败；无法关闭动态窗口才算失败。

## 9. 检查旧需求文档目录

1. 项目根目录应存在英文文件夹 `past_files_can_remove`。
2. 旧需求确认书放在里面。
3. 文件夹中的 README 明确说明可以安全删除。

删除这个文件夹不会影响程序、EXE、设置、壁纸导出和构建过程。验收时可以先保留，不要求实际删除。

## 10. 把失败结果发回来

若某项失败，请按这个格式发送：

```text
失败章节：例如第 6 节“最近上一张逻辑”
操作顺序：A -> 下一张 B -> 下一张 C -> 上一张
实际结果：回到了 A
预期结果：应该回到 B
是否等待桌面切换完成：是 / 否
截图：附上程序界面和当前桌面截图
```

如果所有步骤都通过，只需要回复：

```text
V1 最新反馈验收全部通过。
```
